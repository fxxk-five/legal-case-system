| 编号 | 决策 |

| 11 | 案号支持手动输入；未输入则自动生成。 |
| 12 | 登录入口：仅 1 个案件直达详情；多个案件进入案件列表。 |

## 2. 二次审查结论（基于代码）

## 2.1 可复用能力

- 鉴权与租户边界已具备基础能力（JWT、角色、跨租户拦截）。
- 案件、文件、AI 任务、邀请链路已有可运行实现。
- Web 与小程序已有案件详情、上传、AI 状态展示基础页面。

## 2.2 必须补齐的 P0 缺口

- 缺少 `case_flows` 表及可见性字段 `visible_to`。
- 缺少文件维度字段：`uploader_role`、`description`、`parse_status`。
- 缺少案件维度字段：`analysis_status`、`analysis_progress`。
- 缺少报告接口与 PDF 生成链路（当前尚无 `/cases/{id}/report`）。
- 当前 WS 禁止 client 订阅进度，与决策 #6 冲突，需改造。
- 当前 AI 主要为进程内线程执行，需升级为 DB 队列 + 独立 worker 消费模型。
- 缺少 AI 成本熔断：月预算上限、单案件预算上限、超额降级/停用策略。

## 2.3 当前落地进度（2026-03-22）

- 已完成：`BE07`、`BE08`、`BE09`、`BE10`、`BE11`、`BE12`、`FEM07`。
- 下一项：`QA02`（主链路 E2E 回归）与 `FEM01/FEM04/FEM05`（当事人体验收口）。
- 测试现状：`pytest backend/tests -q` 全绿（93 passed）。
- 生产回归脚本：`scripts/smoke_core_chain.py`、`scripts/smoke_report_visibility.py`。
- 本地联调现状：`docker compose` 已启动，`smoke_core_chain` 与 `smoke_report_visibility` 实跑通过。
