# 法律案件AI辅助系统 (Legal Case AI System)

本项目是一个专为律师和当事人设计的智能化案件管理与分析平台。核心理念是**“极简操作，AI代劳”**：律师只需管理案件，当事人只需上传材料，繁琐的材料解析、事实提取、法律分析和证据核实全部由后台AI自动化流水线完成。

## 1. 核心执行框架

系统的工作流被设计为高度自动化的流水线，最大程度减少人工干预：

1.  **律师建案与邀请**：律师在Web端或小程序创建案件，填写基本信息，并生成邀请链接/二维码（预留微信便捷邀请通道）。
2.  **当事人上传材料**：当事人通过微信小程序接受邀请，并**批量上传**案件相关的图片、文档等材料。
3.  **AI 自动流水线触发**：系统监听到材料批量上传完成后，**自动**触发后台AI处理流程，无需律师手动点击。
4.  **AI 深度处理**：
    *   **向量化 (RAG)**：材料自动进入RAG系统进行切片和向量化存储。
    *   **文档解析**：AI自动提取材料中的关键事实。
    *   **法律分析**：AI基于提取的事实进行法律风险和策略分析。
    *   **证据核实**：AI对事实和证据进行交叉验证与证伪。
5.  **律师查看结果**：律师在Web端直接查看结构化的AI分析报告。
6.  **增量更新**：如果当事人后续补充上传新材料，系统会自动结合新旧材料，重新运行AI流水线并更新结果。

本项目采用前后端分离架构，包含以下主要模块：

```text

│   │   ├── api/             # API 路由定义 (RESTful)
│   │   ├── core/            # 核心配置、安全、错误处理
│   │   ├── db/              # 数据库连接与会话管理
│   │   ├── models/          # SQLAlchemy 数据库模型
│   │   ├── repositories/    # 数据访问层 (DAO)
│   │   ├── schemas/         # Pydantic 数据校验模型
│   │   └── services/        # 核心业务逻辑与 AI 编排
│   ├── alembic/             # 数据库迁移脚本
│   └── tests/               # 单元与集成测试
```

## 3. 角色与端职责划分

*   **律师 (Web端为主)**：系统的主要使用者。Web端作为“管理控制台”和“结果看板”，律师在此进行案件的宏观管理，并直接审阅AI生成的最终分析报告。
*   **当事人 (小程序端)**：系统的数据提供者。小程序端提供极简的交互，方便当事人随时随地批量上传案件材料。
*   **系统后台 (Backend)**：系统的“大脑”。负责处理业务逻辑、调度AI任务、管理数据存储，并向各端提供稳定的API接口。
