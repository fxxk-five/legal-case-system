# Day 26 检查清单：当事人邀请机制

## 今日目标

- 给律师提供邀请入口
- 给当事人提供扫码进入案件的路径

## 已完成内容

- 新增接口 `GET /api/v1/cases/{id}/invite-qrcode`
- 后端会生成 7 天有效的案件邀请 token
- 返回结果包含：
  - `case_id`
  - `tenant_id`
  - `token`
  - `path`

## 当前设计

- 现在优先返回路径和 token，不强依赖微信码图片
- 这样本地开发、接口联调、前后端排障都更简单
- 后续若接入真实微信小程序码接口，可以在此基础上扩展

## 上线前要补

- 接微信小程序码生成接口
- 增加失效、刷新、撤销逻辑
- 记录邀请审计日志
