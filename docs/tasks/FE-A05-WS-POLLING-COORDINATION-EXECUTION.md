# FE-A05 执行单：WS + 轮询协同策略

## 1. 任务目标
- 建立 AI 任务“WS 优先 + 轮询兜底”的统一策略，提升实时性与稳定性。
- 对齐后端 WS 鉴权方案（BE-A01）与任务状态真源（`GET /ai/tasks/{task_id}`）。
- 保持现有页面交互不变，仅改接口协同逻辑。

## 2. 现状基线（代码事实）
- `useWebSocket.js` 仅做基础连接，无 token 注入、无自动重连、无退避策略。
- `useAITask.js` WS 路径未携带鉴权参数；轮询固定 2 秒，无超时截止。
- WS 失败时缺少明确降级状态标记，页面难以识别“实时通道不可用”。

## 3. 目标协同策略
- 优先级：WS 事件 > 轮询结果。
- 真源：任务终态以 `GET /ai/tasks/{task_id}` 为准。
- 降级：
  - WS 连接失败/鉴权失败 -> 自动启动轮询。
  - WS 恢复后 -> 轮询降频或停止。
- 重试：
  - WS 重连指数退避：1s/2s/5s（最多 N 次）。
  - 轮询间隔动态：处理中 2s，接近超时可加大到 3~5s。

## 4. 与后端对齐要点
- WS 鉴权默认方案：`/ws/ai/tasks/{task_id}?token=<jwt>`。
- 若后端返回 failed 事件，前端需保留错误信息并停止轮询。
- 轮询接口 `GET /ai/tasks/{task_id}` 必须能处理 `pending/processing/completed/failed` 全状态。

## 5. 可分配子任务（开发拆分）
| 子任务 | 说明 | 目标目录 | 产出 |
|---|---|---|---|
| A05-1 | WS URL 构建与 token 注入 | `src/composables/useWebSocket.js` | 鉴权可用连接 |
| A05-2 | WS 自动重连与退避 | `useWebSocket.js` | 重连策略 |
| A05-3 | 轮询策略抽象 | `src/composables/useAITask.js` | 动态轮询与超时控制 |
| A05-4 | 协同状态机实现 | `useAITask.js` | ws/polling/degraded 状态 |
| A05-5 | 页面级提示映射 | `TaskProgress.vue` | 降级场景提示（不改样式） |

## 6. 验收标准（DoD）
- WS 可连通时实时更新进度，终态后停止轮询。
- WS 不可用时自动降级轮询，不影响任务完成链路。
- 网络抖动后可自动重连或稳定降级，不出现无限错误弹窗。
- 任务终态不重复写入，不出现状态回退。

## 7. 风险与默认策略
- 风险：WS 鉴权变更后旧逻辑全失败。  
  默认策略：轮询始终保留兜底。
- 风险：过于频繁轮询造成压力。  
  默认策略：动态间隔 + 最大轮询时长上限。

## 8. 回滚策略
1. 出现兼容问题时可临时关闭 WS，仅保留轮询。
2. 保留旧接口调用路径，逐步启用新协同策略。
