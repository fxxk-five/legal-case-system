# FE-A10 执行单：联调回归用例清单

## 1. 任务目标
- 建立 Web + 小程序 + 后端联调回归最小集，覆盖“登录、案件、文件、AI 三链路”。
- 为每次接口调整提供可重复执行的验收路径。
- 仅定义测试步骤与判定标准，不涉及 UI 改动。

## 2. 测试前置
- 后端服务可访问（`/api/v1/health` 正常）。
- 准备至少两类账号：
  - 律师/租户管理员账号
  - 当事人账号（或可创建）
- 准备测试案件与可上传文件（pdf/docx/jpg 各一）。

## 3. 回归用例（核心链路）

## 3.1 认证链路
1. Web 登录成功，进入首页。
2. Web 错误密码返回稳定错误提示。
3. 小程序登录/绑定流程可完成。
4. token 失效后，双端均触发重新登录。

## 3.2 案件链路
1. Web 案件列表加载成功。
2. 创建案件成功并可查看详情。
3. 案件详情更新状态成功。
4. 邀请路径生成成功（`/cases/{id}/invite-qrcode`）。

## 3.3 文件链路
1. Web 上传文件成功并刷新列表。
2. 小程序上传文件成功并刷新列表。
3. 文件下载（直连/短链）成功。
4. 越权用户访问他租户文件被拒绝。

## 3.4 AI 链路
1. 文档解析：发起 -> 进度 -> 事实列表可见。
2. 法律分析：发起 -> 进度 -> 分析结果可见。
3. 证伪验证：发起 -> 进度 -> 证伪记录可见。
4. WS 失败时，轮询可自动兜底并最终收敛。

## 3.5 错误码链路
1. 401 场景提示与跳转一致。
2. 403/404/422 场景提示可读且一致。
3. AI 任务不存在时，双端提示一致。

## 6. 验收标准（DoD）
- 回归清单可由非开发同学按步骤执行。
- 每个失败用例可定位到接口与 `request_id`。
- AI 三链路回归结果可稳定复现。
- 执行清单与当前契约版本同步。

## 7. 风险与默认策略
- 风险：清单过长导致执行成本高。  
  默认策略：区分“最小必跑集”与“扩展回归集”。
- 风险：环境差异导致误判。  
  默认策略：记录环境变量与版本信息，失败先复核环境。

## 8. 回滚策略
1. 回归失败时禁止推进发布，按失败链路回退最近接口变更。
2. 回滚后优先重跑最小必跑集确认恢复。
